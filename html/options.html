PagerDuty Notifications Configuration

1. Account Settings

Provide the account subdomain, and optionally a read/write API key for your account. (If you do not provide an API key, then you will only receive notifications if your session is still active for your subdomain. If you provide a read-only API key, you will still get notifications, but you will be unable to acknowledge or resolve them from the notification popup. You can remove the action buttons below if you prefer to use read-only credentials).

If your PagerDuty console is at https://example.pagerduty.com, then you would put "example".
Optionally provide an API key your account. Should be 20 characters.

2. Notification Settings

Define how often you want to check for new incidents, and additional settings regarding what notifications you wish to receive.

Minimum of 15 seconds.
If you only want to use a read-only API key, then you can select this option to remove the action buttons.

3. Save

Make sure to save your settings if you update anything.
